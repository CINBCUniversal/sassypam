Welcome to Sassypam

This is everything you need to get started building UI at NBCU Ad Sales. Sassypam is broken down into elements, components, modules, and utilities. Simply copy and paste the markup, and you're well on your way to making an app visually cohesive with the rest of NBCU Ad Sales products.

Best Practices
  • You don't have to write CSS anymore! (Unless in the rare cases where you absolutely need to ☺ )
  • Use helper classes to get the design the way you want (ex: whitespace classes, emphasis classes)
  • If you need something that Sassypam doesn't have, don't be afraid to submit a feature request using Github
  • Sassypam is just a guide, do what's best for your app/product/feature!

How To Use This Styleguide
Once you have downloaded/included Sassypam into your project, you can start building. We've grouped the styleguide into four parts.
Utilities - These are the smallest pieces you can use. They include icons, borders, whitespace classes, and colors. Think of them as modifiers. You can use these in conjunction with any HTML tag and/or any other UI element you find here in the styleguide.
Elements - These are the building blocks of apps. They cover typography, buttons, badges, and various form elements.
Components - Components are common, reusable pieces of UI. They are often built up with elements and modifiers, you can get an idea of how OOCSS works by taking a peek at them. You can find search and filter bars, empty states, messages, and pagination in this section.
Modules - These are the the most complex pieces of UI. Modules define large sections of an application and can be tricky to wire up. Some examples are comment boxes, modals, and left hand navigation
Layout - This is where grids, layout helpers, visibility, and positioning classes are located.
